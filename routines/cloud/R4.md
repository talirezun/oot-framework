# R4 — Quarterly Long-Tail Settlement (cloud)

**Trigger:** Claude Code Routine, 1st of quarter 09:00.

**Operational state location:** `firm/excel/reward-species-declaration.xlsx` (X2) Long_Tail_Schedule across all partner sheets; per-partner long-tail statement markdown at `firm/partners/<id>/long-tail-statements/{{LAST_QUARTER}}.md`. All in the Ledger per ADR-001.

## Prompt body

```
You are running the quarterly long-tail entitlement settlement for an ØØT organisation. The quarter just ended is {{LAST_QUARTER}}.

Your task: for every output that has an entry in any partner's reward-species-declaration.xlsx Long_Tail_Schedule with settlement_period='quarterly', compute the partner's long-tail payment for the quarter.

1. Read the firm's reward-species-declaration.xlsx. In Long_Tail_Schedule (one shared sheet keyed by the leading partner_id in column A — ADR-005), extract every row where end_date is empty OR > {{LAST_QUARTER_END}}, AND start_date <= {{LAST_QUARTER_END}}. Settle per partner by grouping on column A (partner_id), NOT by assuming per-partner sheets. (After the ADR-005 shift: partner_share_pct is column D, total_settled_to_date is column H.)

2. For each such (partner, output) pair, query the Ledger or financial system for the realised outcome attributable to the output during {{LAST_QUARTER}}: revenue generated, cost saved, customer impact metric — the metric committed to in the Output Spec.

3. Compute the partner's long-tail payment for the quarter as: outcome_attributable × partner_share_pct / 100.

4. Update the partner's Long_Tail_Schedule sheet: append the quarter to total_settled_to_date.

5. Generate a per-partner long-tail statement at firm/partners/{{partner_id}}/long-tail-statements/{{LAST_QUARTER}}.md following the template at templates/brain/long-tail-statement.md. Include any outcome-attribution caveats (e.g., revenue divergence vs. plan, ambiguous attribution cases).

6. Send statement via email. Founder approval required before payment.

7. Post summary to Slack #compensation: "Q{{N}} long-tail settlements ready. {{COUNT}} partners. Total: {{TOTAL_EUR}}. Founder approval required."

Implementation (Pattern C, ADR-001):
1. Clone the Ledger.
2. Open X2 with openpyxl; iterate over partner sheets; collect Long_Tail_Schedule rows in scope.
3. Compute per-partner settlement; write updates back to X2 Long_Tail_Schedule.
4. Generate per-partner long-tail statement markdown.
5. Stage + signed-commit + push: `git add firm/excel/reward-species-declaration.xlsx firm/partners/*/long-tail-statements/{{LAST_QUARTER}}.md && git commit -S -m "R4: long-tail settlement for {{LAST_QUARTER}}" && git push origin main`.

Failure handling: if outcome attribution data is missing for an output, flag for founder + affected partner review; do not pay until resolved; defer to next quarter if unresolvable. Push-failure handling per R1.
```

## Expected outputs

- Per-partner long-tail statements.
- Updated X2 Long_Tail_Schedule rows.
- Founder-approval packet.
- Slack `#compensation` summary.
- Signed commit on `main`.
