# R8 — Treasury Runway Update (cloud, OPTIONAL)

**Trigger:** Anthropic Remote Routine, Monday 08:00.

> ⚠️ **OPTIONAL Routine.** Required only for organisations adopting the Unit Fund (Generation 2). Without the Unit Fund, simpler treasury tracking via accountant + bank statements is sufficient.

## Prompt body

```
You are running the weekly treasury runway update for an ØØT organisation. Today is {{TODAY}}.

Your task:
1. Pull current balances from all firm bank accounts and stablecoin wallets via configured banking/blockchain APIs.
2. Compute total_cash_eur (FX-converted to base currency).
3. Read upcoming Obligations from treasury-runway.xlsx — payroll, variable, long-tail, suppliers, due in next 90 days.
4. Compute monthly_burn_average over rolling 3 months from historical Cash_Position deltas (NOT from future Obligations — Obligations forecasts forward; burn is realised outflow).
5. Update treasury-runway.xlsx Runway_Calc sheet with today's snapshot row: total_cash_eur, monthly_burn_average, runway_months, unit_fund_outstanding_units, implied_redemption_value, reserve_coverage_ratio.
6. If runway_months < threshold (default 9 months) OR reserve_coverage_ratio < 1.0, post alert to Slack #treasury and email founder.

Failure handling: if any banking API is unreachable, flag in alert; never silently skip an account.
```

## Expected outputs

- X8 Runway_Calc updated with today's snapshot row.
- Alert posted to `#treasury` if thresholds breached.

## Setup checklist

1. Confirm X8 (`treasury-runway.xlsx`) exists. If not adopting the Unit Fund, **skip R8**.
2. Anthropic Routines dashboard → new Routine, Monday 08:00.
3. Upload prompt body. Attach `finance-treasury` Skill Pack.
4. Configure banking/blockchain API connectors (jurisdiction-specific; may require firm's accountant to provide read-only API tokens).
5. Configure Slack `#treasury` + Email.
6. Manual test fire; verify X8 Runway_Calc row appended.
