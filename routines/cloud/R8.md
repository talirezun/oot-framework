# R8 — Treasury Runway Update (cloud, OPTIONAL)

**Trigger:** Claude Code Routine, Monday 08:00.

**Operational state location:** `firm/excel/treasury-runway.xlsx` (X8) Cash_Position + Runway_Calc sheets, plus a daily markdown snapshot at `firm/treasury/{{TODAY}}.md`, in the Ledger per ADR-001 + ADR-004.

> ⚠️ **OPTIONAL Routine.** Required only for organisations adopting the Unit Fund (Generation 2). Without the Unit Fund, simpler treasury tracking via accountant + bank statements is sufficient.

## Prompt body

```
You are running the weekly treasury runway update for an ØØT organisation. Today is {{TODAY}}.

Your task (appended-row contract, ADR-004 §3 — order matters):
1. Pull current balances from all firm bank accounts and stablecoin wallets via configured banking/blockchain APIs.
2. Append the day's Cash_Position rows FIRST (one per account: date, account_label, balance, currency, balance_eur_equivalent — all literals). Burn is a delta over the just-appended history, so these rows must exist before you compute burn.
3. Compute total_cash_eur (FX-converted to base currency) by summing the day's Cash_Position rows in Python.
4. Read upcoming Obligations from treasury-runway.xlsx — payroll, variable, long-tail, suppliers, due in next 90 days (context only; NOT the burn source).
5. Compute monthly_burn_average over rolling 3 months from historical Cash_Position deltas (NOT from future Obligations — Obligations forecasts forward; burn is realised outflow).
6. Append one Runway_Calc snapshot row. Write literals for snapshot_date (A), total_cash_eur (B), monthly_burn_average (C), unit_fund_outstanding_units (E), implied_redemption_value (F). MUST write the D formula =IFERROR(B{R}/C{R},0) and the G formula =IFERROR(B{R}/IF(F{R}=0,1,F{R}),0) on the appended row.
7. Write a one-paragraph markdown snapshot to firm/treasury/{{TODAY}}.md (runway_months, reserve_coverage_ratio, any breach) so R6's audit trail — which scans Ledger markdown — sees treasury activity (ADR-004 §4). Commit it in the SAME commit as the X8 mutation: git add firm/excel/treasury-runway.xlsx firm/treasury/{{TODAY}}.md && git commit -S && git push origin main.
8. If runway_months < threshold (default 9 months) OR reserve_coverage_ratio < 1.0, post alert to Slack #treasury and email founder.

Failure handling: if any banking API is unreachable, flag in alert; never silently skip an account.
```

## Expected outputs

- X8 Cash_Position + Runway_Calc updated with today's rows (D/G formulas written on the appended Runway_Calc row).
- `firm/treasury/{{TODAY}}.md` markdown snapshot (visible to R6's audit trail).
- One signed commit on `main` carrying both.
- Alert posted to `#treasury` if thresholds breached.

## Setup checklist

1. Confirm X8 (`firm/excel/treasury-runway.xlsx`) exists in the Ledger. If not adopting the Unit Fund, **skip R8**.
2. Claude Code → `/schedule` → new Routine, Monday 08:00.
3. Upload prompt body. Attach `finance-treasury` Skill Pack.
4. Configure banking/blockchain API connectors (jurisdiction-specific; may require firm's accountant to provide read-only API tokens).
5. Configure connectors: GitHub (read + write with signing), Slack `#treasury`, Email.
6. **Confirm code execution is enabled** — required for the X8 openpyxl write.
7. Manual test fire; verify a signed commit lands appending the snapshot row to X8.
