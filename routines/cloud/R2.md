# R2 — Weekly BR Prep (cloud)

**Trigger:** Claude Code Routine, Friday 08:00 in the firm's primary timezone.

**Operational state location:** reads `firm/excel/partner-output-ledger.xlsx` (X1), `firm/excel/klarna-test.xlsx` (X4), `firm/excel/agent-skill-roi.xlsx` (X6), `firm/excel/reward-species-declaration.xlsx` (X2), and `firm/excel/treasury-runway.xlsx` (X8, if adopted); reads + writes `firm/excel/business-review.xlsx` (X3) — all in the Ledger per ADR-001.

> **Known schema gap (fix scheduled, ADR pending):** Decision_Log has no `status` column; the scoring/remediation state machine currently has no Excel home. Do not write a `status` value — column I (`decision`) is formula-driven. Step 3 below reads for tests in `scoring`/`remediation`/`monitoring` states, but those states are not persisted in X4 today; until the ADR lands, source them from the `firm/klarna-tests/{{test_id}}.md` Ledger page frontmatter instead.

## Prompt body

```
You are preparing the Friday Business Review agenda for an ØØT organisation. The BR meeting starts at 14:00 today.

Your task: populate business-review.xlsx (X3) Weekly_Review sheet for the week starting {{MONDAY_DATE}} and ending today.

1. Read partner-output-ledger.xlsx Output_Log for the past 7 days. Identify the top 5 outputs by computed_variable (which is value_envelope × partner_multiplier × IF(rework_within_30d="Yes", 0, 1)). For each: who shipped it, what tier, brief description from the output_spec_ref. Write to column B (notable_outputs) as a markdown bullet list with partner wikilinks.

2. Query the Ledger (firm/output-logs/) for blocker tags raised in the past 7 days that remain open. Query Slack channels for #blocker tags not resolved. Write to column C (blockers).

3. Read klarna-test.xlsx Decision_Log for any tests in scoring | remediation | monitoring state. Write to column D (klarna_test_status). Format: test_id, summary, days-since-trigger, owner.

4. Read the KPI sources for week-over-week deltas and write to column E (kpi_movements) as a small table. Each KPI has a specific home (verify the sheet/column exists before reading):
   - `ai_skill_roi` — agent-skill-roi.xlsx (X6) ROI_Calc sheet, `roi_multiple` column.
   - `agent_human_ratio` — agent-skill-roi.xlsx (X6) Human_Agent_Ratio sheet, `agent_human_ratio` column.
   - `treasury_runway_months` — treasury-runway.xlsx (X8) Runway_Calc sheet, `runway_months` column. X8 is OPTIONAL — if the firm has not adopted the Unit Fund / X8, write "n/a" for this KPI.
   - `partner_count` — reward-species-declaration.xlsx (X2) Partner_Profile sheet (count of partner rows).
   - `customer_count` — business-review.xlsx (X3) Monthly_BR sheet, `customer_count_delta` column (rolled up monthly; use the latest available month).

5. Identify decisions due — items in the Ledger (firm/output-logs/) tagged decision-pending in the past 7 days. Write to column F (decisions_due).

6. Create a pre-meeting Ledger page at firm/business-reviews/{{TODAY}}-pre.md following the template at templates/brain/business-review.md. The page mirrors the X3 row plus has post-meeting sections that the scribe will fill after the BR.

7. Post a draft summary to Slack #business-review by 09:00:
"BR agenda for {{TODAY}} 14:00 ready. Highlights: {{TOP_3_FROM_NOTABLE}}. Klarna in flight: {{COUNT}}. Decisions due: {{COUNT}}. Pre-read at [[business-reviews/{{TODAY}}-pre]]."

Implementation (Pattern C, ADR-001):
1. Clone the Ledger.
2. Open X1, X4, X6, X2, and X8 (if present) with openpyxl; read the data needed for the BR.
3. Open X3 with openpyxl; populate the Weekly_Review row (and read Monthly_BR for `customer_count_delta`).
4. Write the pre-meeting Ledger page.
5. Stage + signed-commit + push: `git add firm/excel/business-review.xlsx firm/business-reviews/{{TODAY}}-pre.md && git commit -S -m "R2: BR prep for week ending {{TODAY}}" && git push origin main`.

After the BR meeting (signaled by the Decisions_Log getting populated by the scribe), commit the final summary by renaming the page from -pre.md to {{TODAY}}.md and setting frontmatter status: active. (This rename + status change is one more signed commit.)

Failure handling: if any data source is unreachable, populate from what's available and clearly mark the gap in column G (meeting_notes) for the BR participants to address. Push-failure handling per R1.
```

## Expected outputs

- `firm/excel/business-review.xlsx` Weekly_Review row populated.
- Ledger page at `firm/business-reviews/{{TODAY}}-pre.md` (pre-meeting); renamed to `{{TODAY}}.md` post-meeting.
- Slack draft summary to `#business-review`.
- Signed commit(s) on `main`.
