# R5 — Brain Health Check (cloud)

**Trigger:** Anthropic Remote Routine, Sunday 09:00.

## Prompt body

```
You are running the weekly Brain health check for an ØØT organisation.

Your task:
1. Run my-curator's scan_wiki_health on every domain (firm, customers, products, legal, research, partners — whatever exists).
2. Compile a summary of issues by type: broken wikilinks, orphan pages, semantic duplicates, stale pages (>90 days no update on pages tagged active).
3. Auto-fix the safe categories — broken wikilinks where the target slug exists with a typo correctable via fix_wiki_issue.
4. For unsafe categories, post a list to Slack #brain-health for human review and fix-this-week tagging.
5. Run scan_semantic_duplicates and post any new duplicate clusters (clusters not previously dismissed via dismiss_wiki_issue).
6. Update firm/brain-health/YYYY-WW.md with the weekly health snapshot.

Failure handling: if Curator is unreachable, retry hourly until 18:00; if still unreachable, escalate to ops via #ops.
```

## Expected outputs

- Weekly Brain health snapshot at `firm/brain-health/YYYY-WW.md`.
- Slack `#brain-health` post listing items requiring human attention.
- Auto-fixes applied for typo-correctable broken wikilinks.

## Setup checklist

1. Confirm Curator MCP is configured and `list_domains` returns the firm's domains.
2. Anthropic Routines dashboard → new Routine, Sunday 09:00.
3. Upload prompt body. Attach `my-curator` Skill Pack.
4. Configure connectors: Slack (post `#brain-health` + `#ops`).
5. Manual test fire on a Saturday; verify the snapshot lands.
