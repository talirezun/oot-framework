# R3 — Monthly Variable Calc (privacy)

**Trigger:** OS-native scheduling, 1st of month 09:00.

> ⚠️ **Use the largest local model available.** Compensation accuracy is high-stakes; Qwen 3 14B is acceptable but Llama 3.3 70B (or equivalent) is preferred. The framework's authors recommend running R3 on a model class up from the firm's daily-driver model.

## launchd plist (macOS)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key><string>oot.r3.monthly-variable-calc</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/llmster</string>
        <string>--model</string><string>llama-3.3-70b-instruct</string>
        <string>--skill</string><string>compensation-attribution</string>
        <string>--prompt-file</string>
        <string>/Users/oot/oot-framework/routines/privacy/r3.prompt.md</string>
    </array>
    <key>StartCalendarInterval</key>
    <dict>
        <key>Day</key><integer>1</integer>
        <key>Hour</key><integer>9</integer>
        <key>Minute</key><integer>0</integer>
    </dict>
</dict>
</plist>
```

## cron (Linux)

```cron
0 9 1 * * /usr/local/bin/llmster --model llama-3.3-70b-instruct --skill compensation-attribution --prompt-file ~/oot-framework/routines/privacy/r3.prompt.md >> ~/oot-framework/logs/r3.log 2>&1
```

## Prompt body

Identical to `routines/cloud/R3.md` with these substitutions:
- Email → 4thtech dMail.
- Slack `#compensation` → 4thtech dChat `#compensation`.
- Excel reads/writes via Excel MCP.

## Setup checklist (privacy)

1. Verify the larger model (e.g. Llama 3.3 70B) is downloaded in LM Studio (~40 GB).
2. Confirm always-on machine RAM is sufficient (32-64 GB for 70B at 4-bit quantisation).
3. R1 has produced ≥30 days of X1 data.
4. At least one partner has a populated X2 sheet.
5. Place prompt at `~/oot-framework/routines/privacy/r3.prompt.md`.
6. Install launchd plist or cron entry.
7. Test on a fresh test sheet before first real run.
