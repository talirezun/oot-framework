**Trigger:** OS-native scheduling, daily 23:00.

> ⚠️ Signing key on the always-on machine is stored in a Trezor or YubiKey-backed slot, never plaintext. `gpg-agent` configured to cache passphrase for 24h.

## Prompt body

Identical to `routines/cloud/R6.md`. The signed-commit discipline is identical; the signing key is on the always-on machine's Trezor/YubiKey-protected slot.

## Setup checklist (privacy)

1. **Configure branch protection** (one-time): same as cloud. Force-push disabled, signed commits required, ≥1 reviewer for `firm/audit-logs/*`.
2. **Configure GPG signing on the always-on machine.** Generate or import the `oot-bot` GPG key. Configure `gpg-agent` for 24h passphrase cache. Test: `git commit -S -m test`.
3. **Trezor/YubiKey-backed signing slot** strongly recommended for production privacy-track instances.
4. Place prompt at `~/oot-framework/routines/privacy/r6.prompt.md`.
5. Install launchd plist or cron entry.
6. Manual test fire; verify a signed commit lands on the protected branch.
