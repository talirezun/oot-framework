**Trigger:** OS-native scheduling on the always-on machine, Friday 08:00.

## cron (Linux)

```cron
0 8 * * 5 /usr/local/bin/llmster --model qwen-3-14b-instruct --skill reporting-business-review --skill my-curator --prompt-file ~/oot-framework/routines/privacy/r2.prompt.md >> ~/oot-framework/logs/r2.log 2>&1
```

## Prompt body

Identical to `routines/cloud/R2.md` with these substitutions:
- Slack `#blocker` → 4thtech dChat `#blocker`.
- Slack `#business-review` post → 4thtech dChat `#business-review`.
- Excel reads/writes: same openpyxl operation against a local Ledger clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Confirm always-on machine + LM Studio MCP host running.
2. Confirm R1 is producing daily output logs (X1 + Brain pages).
3. Place the prompt body at `~/oot-framework/routines/privacy/r2.prompt.md`.
4. Install launchd plist or cron entry per platform.
5. Manual test fire: `launchctl start oot.r2.weekly-br-prep`. Verify X3 + Brain pre-page + dChat post.
