# R2 — Weekly BR Prep (privacy)

**Trigger:** OS-native scheduling on the always-on machine, Friday 08:00.

## launchd plist (macOS)

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key><string>oot.r2.weekly-br-prep</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>-lc</string>
        <string>cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/qwen-3-14b-instruct "$(cat ~/oot-framework/routines/privacy/r2.prompt.md)"</string>
    </array>
    <key>StartCalendarInterval</key>
    <dict>
        <key>Weekday</key><integer>5</integer>
        <key>Hour</key><integer>8</integer>
        <key>Minute</key><integer>0</integer>
    </dict>
</dict>
</plist>
```

## cron (Linux)

```cron
0 8 * * 5 cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/qwen-3-14b-instruct "$(cat ~/oot-framework/routines/privacy/r2.prompt.md)" >> ~/oot-framework/logs/r2.log 2>&1
```

> Harness = OpenCode (`opencode run`); model server = local LM Studio hosted by the `llmster` daemon; the `cd ~/{{FIRM_SLUG}}` runner directory carries the scoped `opencode.json` allow-list (the unattended exception). See R1's note and [`OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md) → "Scheduled / unattended runs".

## Prompt body

Identical to `routines/cloud/R2.md` with these substitutions:
- Slack `#blocker` → 4thtech dChat `#blocker`.
- Slack `#business-review` post → 4thtech dChat `#business-review`.
- Excel reads/writes: same openpyxl operation against a local Ledger clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Confirm the model stack is up: LM Studio + `llmster` daemon serving on `http://127.0.0.1:1234/v1`, model warm (`lms load qwen-3-14b-instruct --ttl 3600`), OpenCode installed with the `lmstudio` provider and MCP servers configured (see R1 setup).
2. Confirm R1 is producing daily output logs (X1 + Brain pages).
3. Place the prompt body at `~/oot-framework/routines/privacy/r2.prompt.md` (it opens by instructing the agent to read the reporting-business-review + my-curator SKILL.md files).
4. Install launchd plist or cron entry per platform.
5. Manual test fire: `launchctl start oot.r2.weekly-br-prep`. Verify X3 + Brain pre-page + dChat post.
