# R8 — Treasury Runway Update (privacy, OPTIONAL)

**Trigger:** OS-native scheduling, Monday 08:00.

> ⚠️ **OPTIONAL Routine.** Required only for organisations adopting the Unit Fund (Generation 2).

## cron (Linux)

```cron
0 8 * * 1 cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/qwen-3-14b-instruct "$(cat ~/oot-framework/routines/privacy/r8.prompt.md)" >> ~/oot-framework/logs/r8.log 2>&1
```

> Harness = OpenCode (`opencode run`); model server = local LM Studio hosted by the `llmster` daemon; the `cd ~/{{FIRM_SLUG}}` runner directory carries the scoped `opencode.json` allow-list (the unattended exception). See R1's note and [`OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md) → "Scheduled / unattended runs".

## Prompt body

Identical to `routines/cloud/R8.md` with these substitutions:
- Email → 4thtech dMail.
- Slack `#treasury` → 4thtech dChat `#treasury`.
- Excel writeback: same openpyxl operation against a local Ledger clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Confirm X8 exists. If not adopting Unit Fund, skip R8.
2. Configure banking API access (jurisdiction-specific; most banks expose HTTP APIs that work identically on cloud + privacy track).
3. Place prompt at `~/oot-framework/routines/privacy/r8.prompt.md`.
4. Install launchd plist or cron entry.
5. Manual test fire.
