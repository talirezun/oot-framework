# R7 — Klarna Test Trigger (privacy)

**Trigger:** Local GitHub-webhook listener on the always-on machine, polled every 5 minutes for new `ai-replaces-human` PR labels.

> ⚠️ **Use the largest local model available.** Klarna Test scoring affects compensation discipline + framework reputation; do not run on small models without careful evaluation.

## Local listener setup

A small Python service runs as a launchd / systemd unit on the always-on machine:

```python
# ~/oot-framework/installer/privacy/r7_listener.py
# Polls GitHub PRs with label `ai-replaces-human` every 5 min.
# On a new label-applied event, invokes llmster with the R7 prompt.
```

Install:

```cron
*/5 * * * * /usr/bin/python3 ~/oot-framework/installer/privacy/r7_listener.py >> ~/oot-framework/logs/r7-listener.log 2>&1
```

The listener also exposes a manual-trigger flag for partners who want to start a Klarna Test from the command line.

## Prompt body

Identical to `routines/cloud/R7.md` with these substitutions:
- Email → 4thtech dMail.
- Slack `#klarna-test` post → 4thtech dChat `#klarna-test`.
- Excel writeback: same openpyxl operation against a local Ledger clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Configure GitHub Actions workflow `.github/workflows/klarna-gate.yml` (same as cloud — branch protection enforces).
2. Configure auto-labeller `.github/labeler.yml` (same as cloud).
3. Install the local listener service: `python3 -m installer.privacy.r7_listener --install`.
4. Verify the listener runs every 5 min: `ps aux | grep r7_listener`.
5. Test: open a draft PR; apply `ai-replaces-human` label; within 5 min, dChat `#klarna-test` posts.
