# R4 — Quarterly Long-Tail Settlement (privacy)

**Trigger:** OS-native scheduling, 1st of quarter 09:00.

## cron (Linux)

```cron
0 9 1 1,4,7,10 * /usr/local/bin/llmster --model llama-3.3-70b-instruct --skill compensation-attribution --skill finance-treasury --prompt-file ~/oot-framework/routines/privacy/r4.prompt.md >> ~/oot-framework/logs/r4.log 2>&1
```

## launchd plist (macOS)

Use a single plist with `StartCalendarInterval` array of 4 entries (Jan 1, Apr 1, Jul 1, Oct 1 at 09:00). See `examples/launchd/r4.plist`.

## Prompt body

Identical to `routines/cloud/R4.md` with these substitutions:
- Email → 4thtech dMail.
- Slack `#compensation` → 4thtech dChat `#compensation`.
- Excel writeback: same openpyxl operation against a local Brain-repo clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Confirm at least one X2 Long_Tail_Schedule entry.
2. Place prompt at `~/oot-framework/routines/privacy/r4.prompt.md`.
3. Install launchd plist or cron entry.
4. Manual test fire on a non-quarter date with `--dry-run` flag (writes to a test path).
