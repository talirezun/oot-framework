# R4 — Quarterly Long-Tail Settlement (privacy)

**Trigger:** OS-native scheduling, 1st of quarter 09:00.

## cron (Linux)

```cron
# The `lms unload --all; lms load … --ttl 900 -y &&` prefix is the robust scheduled-fire pattern (OPENCODE-SETUP.md §h, TTL/JIT trap).
0 9 1 1,4,7,10 * cd ~/{{FIRM_SLUG}} && lms unload --all; lms load llama-3.3-70b-instruct --context-length 32768 --ttl 900 -y && /usr/local/bin/opencode run --model lmstudio/llama-3.3-70b-instruct "$(cat ~/oot-framework/routines/privacy/r4.prompt.md)" >> ~/oot-framework/logs/r4.log 2>&1
```

> Harness = OpenCode (`opencode run`); model server = local LM Studio, served headless (the LM Studio desktop app's own server on macOS, or the standalone `llmster` daemon on a headless Linux box; `lms load llama-3.3-70b-instruct --context-length 32768 --ttl 3600` — the `--context-length` is required, the default 4096 can't hold the agent's system prompt); the `cd ~/{{FIRM_SLUG}}` runner directory carries the scoped `opencode.json` allow-list (the unattended exception). See R1's note and [`OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md) → "Scheduled / unattended runs".

## launchd plist (macOS)

Use a single plist with `StartCalendarInterval` as an **array** of 4 entries (Jan 1, Apr 1, Jul 1, Oct 1 at 09:00). The plist file (`examples/launchd/r4.plist`, ships as a file in v1.x) is authored at install time by the founder's agent following these inline fields:

- `Label` = `oot.r4.quarterly-long-tail`
- `ProgramArguments` = `["/bin/bash", "-lc", "cd ~/{{FIRM_SLUG}} && lms unload --all; lms load llama-3.3-70b-instruct --context-length 32768 --ttl 900 -y && /usr/local/bin/opencode run --model lmstudio/llama-3.3-70b-instruct \"$(cat ~/oot-framework/routines/privacy/r4.prompt.md)\""]` — the same OpenCode invocation as the cron line above (including the `lms unload --all; lms load …` scheduled-fire prefix), wrapped in `bash -lc` so the `$(cat …)` prompt substitution and the `~` expansion work under launchd
- `StartCalendarInterval` = an array of 4 dicts, each with `Month` ∈ {1,4,7,10}, `Day` = 1, `Hour` = 9, `Minute` = 0
- `StandardOutPath` / `StandardErrorPath` = `~/oot-framework/logs/r4.log` / `r4.err`

Install: `launchctl load ~/Library/LaunchAgents/oot.r4.plist`.

## Prompt body

Identical to `routines/cloud/R4.md` with these substitutions:
- Email → 4thtech dMail.
- Slack `#compensation` → 4thtech dChat `#compensation`.
- Excel writeback: same openpyxl operation against a local Ledger clone (per ADR-001; no Excel MCP needed for the Routine path).

## Setup checklist (privacy)

1. Confirm at least one X2 Long_Tail_Schedule entry.
2. Place prompt at `~/oot-framework/routines/privacy/r4.prompt.md` (it opens by instructing the agent to read the compensation-attribution + finance-treasury SKILL.md files).
3. Install launchd plist or cron entry.
4. Manual test fire on a non-quarter date by pointing the prompt at a throwaway test Ledger clone (edit the repo path at the top of `r4.prompt.md`) so no real settlement is written — then run `opencode run --model lmstudio/llama-3.3-70b-instruct "$(cat ~/oot-framework/routines/privacy/r4.prompt.md)"` by hand and inspect the result.
