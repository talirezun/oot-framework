# R4 ã Quarterly Long-Tail Settlement (privacy)

**Trigger:** OS-native scheduling, 1st of quarter 09:00.

## cron (Linux)

```cron
0 9 1 1,4,7,10 * cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/llama-3.3-70b-instruct "$(cat ~/oot-framework/routines/privacy/r4.prompt.md)" >> ~/oot-framework/logs/r4.log 2>&1
```

> Harness = OpenCode (`opencode run`); model server = local LM Studio hosted by the `llmster` daemon (`lms load llama-3.3-70b-instruct --ttl 3600`); the `cd ~/{{FIRM_SLUG}}` runner directory carries the scoped `opencode.json` allow-list (the unattended exception). See R1's note and [`OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md) ã "Scheduled / unattended runs".

## launchd plist (macOS)

Use a single plist with `StartCalendarInterval` as an **array** of 4 entries (Jan 1, Apr 1, Jul 1, Oct 1 at 09:00). The plist file (`examples/launchd/r4.plist`, ships as a file in v1.x) is authored at install time by the founder's agent following these inline fields:

- `Label` = `oot.r4.quarterly-long-tail`
- `ProgramArguments` = `["/bin/bash", "-lc", "cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/llama-3.3-70b-instruct \"$(cat ~/oot-framework/routines/privacy/r4.prompt.md)\""]` ã the same OpenCode invocation as the cron line above, wrapped in `bash -lc` so the `$(cat ãÎ)` prompt substitution and the `~` expansion work under launchd
- `StartCalendarInterval` = an array of 4 dicts, each with `Month` ã {1,4,7,10}, `Day` = 1, `Hour` = 9, `Minute` = 0
- `StandardOutPath` / `StandardErrorPath` = `~/oot-framework/logs/r4.log` / `r4.err`

Install: `launchctl load ~/Library/LaunchAgents/oot.r4.plist`.
