# R1 — Daily Output Capture (privacy)

**Trigger:** OS-native scheduling on the always-on machine, daily 18:00 (firm timezone).

## launchd plist (macOS)

```xml
<!-- ~/Library/LaunchAgents/oot.r1.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key><string>oot.r1.daily-output-capture</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>-lc</string>
        <string>cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/qwen-3-14b-instruct "$(cat ~/oot-framework/routines/privacy/r1.prompt.md)"</string>
    </array>
    <key>StartCalendarInterval</key>
    <dict><key>Hour</key><integer>18</integer><key>Minute</key><integer>0</integer></dict>
    <key>StandardOutPath</key><string>/Users/oot/oot-framework/logs/r1.log</string>
    <key>StandardErrorPath</key><string>/Users/oot/oot-framework/logs/r1.err</string>
</dict>
</plist>
```

Install: `launchctl load ~/Library/LaunchAgents/oot.r1.plist`. Verify: `launchctl list | grep oot.r1`.

## cron (Linux)

```cron
# R1 daily output capture — 18:00
0 18 * * * cd ~/{{FIRM_SLUG}} && /usr/local/bin/opencode run --model lmstudio/qwen-3-14b-instruct "$(cat ~/oot-framework/routines/privacy/r1.prompt.md)" >> ~/oot-framework/logs/r1.log 2>&1
```

Install: `crontab -e`, paste, save. Verify: `crontab -l`.

> **Harness, model server, and unattended permissions.** The command is **OpenCode** (`opencode run`, non-interactive) — the agent that actually clones the Ledger, runs openpyxl, and calls MCP servers. It talks to a **local LM Studio server** (started headless by the `llmster` daemon and kept warm with `lms load qwen-3-14b-instruct --ttl 3600`), configured in `opencode.json` as an OpenAI-compatible provider conventionally named `lmstudio`. Because `cd ~/{{FIRM_SLUG}}` puts OpenCode in the firm's runner directory, it picks up that directory's **project-level `opencode.json`, which carries a scoped allow-list** (bash patterns for `git`/`python3`/`mktemp`/openpyxl work + edit within the workdir; everything else `ask`). This is the documented **unattended exception** to the install-time ask-everything rule — cron cannot answer an interactive "ask" prompt, so the runner directory pre-authorises exactly the operations R1 needs and nothing more. Do **not** use OpenCode's global `--auto` auto-approve for this; the scoped project config is the safe form. See [`installer/agent-assisted/OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md) → "Scheduled / unattended runs".

## Task Scheduler (Windows)

The XML template (`examples/task-scheduler/r1.xml`, ships as a file in v1.x) is authored at install time by the founder's agent. The simplest path is to skip the XML entirely and register the daily 18:00 task directly:

```
schtasks /create /tn "OOT R1 Daily Output Capture" /tr "python C:\oot-framework\routines\privacy\run_r1.py" /sc DAILY /st 18:00
```

If you prefer an XML for version control, export it after creating the task with `schtasks /query /tn "OOT R1 Daily Output Capture" /xml > r1.xml`, then re-import on other machines with `schtasks /create /xml "r1.xml" /tn "OOT R1 Daily Output Capture"`.

## Prompt body

Identical to the cloud version (`routines/cloud/R1.md`), with these source substitutions:
- GitHub: via GitHub MCP (PAT from secrets store).
- Slack channels → 4thtech dChat threads (`#output`, `#commercial`, `#sales`).
- Google Workspace read → local filesystem via Desktop Commander MCP, scanning `{{TRACKED_FOLDERS}}` paths.
- Slack post target → 4thtech dChat `#output-log`.
- Excel writeback: same openpyxl operation against a local Ledger clone — no Excel MCP needed for the Routine path. (Excel MCP remains optional for ad-hoc human-in-the-loop work.)

The full prompt body is identical otherwise (the deterministic rework-detection rule, the X1 cross-workbook discipline, the Brain daily-log discipline). Per ADR-001, both cloud and privacy tracks perform the same openpyxl + signed-commit operation; only the substrate differs.

## Expected outputs

- Rows appended to `firm/excel/partner-output-ledger.xlsx` in the local Ledger clone, signed-committed and pushed to `main`.
- Ledger page at `firm/output-logs/YYYY-MM-DD.md`.
- 4thtech dChat post to `#output-log`.

## Setup checklist (privacy track)

1. Verify the always-on machine is configured (Mac mini / NUC / Pi 5 with FDE + UPS) per Skill Pack S12 §4.2.
2. Install the model-serving stack: **LM Studio ≥0.3.5** with the **llmster** headless daemon (per [lmstudio.ai/docs/developer/core/headless](https://lmstudio.ai/docs/developer/core/headless)) + the **`lms`** CLI (LM Studio's own CLI: `lms server start`, then `lms load qwen-3-14b-instruct --ttl 3600` to keep the model warm). Then install **OpenCode** as the agent harness and point its `lmstudio` provider at `http://127.0.0.1:1234/v1` — see [`installer/agent-assisted/OPENCODE-SETUP.md`](../../installer/agent-assisted/OPENCODE-SETUP.md).
3. Configure MCP servers (github-mcp, desktop-commander, 4thtech) in the runner directory's `opencode.json` `mcp` block. Excel MCP is **optional** — the Routine itself uses openpyxl directly via OpenCode's code execution.
4. Place the prompt file at `~/oot-framework/routines/privacy/r1.prompt.md` (separate file; it begins with an instruction to read the named SKILL.md file(s) first, replacing the old `--skill` flags).
5. Create `~/oot-framework/logs/` directory (`mkdir -p ~/oot-framework/logs`).
6. Install the launchd plist (macOS), cron entry (Linux), or Task Scheduler XML (Windows) per platform.
7. Manual test fire: `launchctl start oot.r1.daily-output-capture` (or equivalent). Verify X1 row + Brain page + dChat post.
