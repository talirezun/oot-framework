# R5 — Brain Health Check (privacy)

**Trigger:** OS-native scheduling, Sunday 09:00.

**No bridge needed.** Unlike the cloud-track R5 (which uses Curator's GitHub sync to reach the Second Brain — see [`routines/cloud/R5.md`](../cloud/R5.md) and [`docs/AUTOMATION-PIPELINE.md`](../../docs/AUTOMATION-PIPELINE.md) § "How the bridge works"), privacy-track R5 runs on the same always-on machine as the my-curator MCP server. It calls the 17 MCP tools directly — full `scan_wiki_health`, `scan_semantic_duplicates`, `fix_wiki_issue` are all available. Auto-fix of typo-correctable wikilinks works here, where it cannot on cloud track.

## cron (Linux)

```cron
0 9 * * 0 /usr/local/bin/llmster --model qwen-3-14b-instruct --skill my-curator --prompt-file ~/oot-framework/routines/privacy/r5.prompt.md >> ~/oot-framework/logs/r5.log 2>&1
```

## Prompt body

Identical to `routines/cloud/R5.md` with one substitution:
- Slack `#brain-health` post → 4thtech dChat `#brain-health`.

## Setup checklist (privacy)

1. Confirm Curator MCP is reachable from LM Studio MCP host.
2. Place prompt at `~/oot-framework/routines/privacy/r5.prompt.md`.
3. Install launchd plist or cron entry.
4. Manual test fire (any time of week); verify the snapshot Brain page lands.
