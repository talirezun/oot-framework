# Three Philosophies, One Goal: Augment Code, Claude Code, Codex CLI

**By Dr. Tali Režun**
Vice Dean of Frontier Technologies, COTRUGLI Business School

> The selection rubric for the three coding agents that matter — Augment Code (large multi-file builds), Claude Code (daily development + Plan Mode), Codex CLI (review/audit) — and how to choose between them rather than collapse into one.

---

## Summary

*A condensed summary will land here in v1.x. The full article is at the external link above.*

The article frames the three leading coding agents not as competitors but as **three different philosophies of human–AI collaboration**, each strongest in a different mode of work. Augment Code wins on indexed, multi-file builds where the model needs the whole repository in working memory. Claude Code wins on daily development with Plan Mode and tight feedback loops. Codex CLI wins as the dispassionate reviewer / auditor in the loop. Trying to use one tool for all three jobs — the common temptation — produces worse output across the board than picking the right tool per task. This is the basis for the rubric in Skill Pack S4 §4.4.

## Why this matters for ØØT

Thesis 2 (centaur work) makes a strong claim that humans + AI must collaborate at the *task* level, not the *role* level. The three-philosophies rubric is what operationalises that claim for software engineering inside the firm. It also feeds the partner-output discipline: a partner's output is graded by the artefact, not by which agent produced it.
